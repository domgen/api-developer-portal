Authorizations:
BearerAuth
header Parameters
X-Request-Id
required
string (X-Request-Id) ^[0-9a-fA-F]{8}\b-[0-9a-fA-F]{4}\b-[0-9a-fA-F...
Example: 96113c99-304f-44f2-b3a2-4a3f301ce148

uuidv4 string to identify the individual request

Request Body schema: application/json
lineItemId
required
string [ 1 .. 64 ]

The Id of the individual line item within the purchase. This id is used when updating or retrieving information around an individual policy in follow-up API calls after the initial ITB has been registered.

reasonCode
required
string [ 11 .. 12 ]
Enum: "PREDELIVERY" "POSTDELIVERY" "NOTREQUIRED"

A Domestic & General defined reason code for the cancellation:

  • PREDELIVERY: Policy cancelled before appliance delivered
  • POSTDELIVERY: Policy cancelled after appliance delivered
  • NOTREQUIRED: Policy no longer required
reasonReference
string [ 1 .. 50 ]

A client defined reason code for the cancellation used for performance reporting only.

correspondanceCode
string [ 5 .. 11 ]
Enum: "EMAIL" "LETTER" "NOTREQUIRED"

A Domestic & General defined Corerespondance code for the cancellation:

  • EMAIL:
  • LETTER:
  • NOTREQUIRED:
